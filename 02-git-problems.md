#Git.Проблемы

Вначале писал gti, вместо git. Думаю эта проблема на долго
С git commit лучше ВСЕГДА писать -m "Сообщение коммита", иначе вылезает страшный консольный интерфейс, которые тяжко закрывать
git remote add origin - начинает отслеживать репозиторий
НЕ ЗАБЫТЬ переименовать master ветку в main
При git push, нужно в первый раз написать -u origin main
При пуше иногда может мешать VPN или прокси. Просто потыкать их типа включить или выключит, мне помогло

ВАЖНОЕ: Нужно было настроить SSH-ключ
при помощи ssh-keygen -t ed25519 -C "Моя почта"
Потом узнаём этот ключ при помощи cat ~/.ssh/id_ed25519.pub
И после этого всего надо перепривязать репозиторий. По умолчанию он по ссылке типа  https://github.com/...
меняем его git remote set-url origin git@github.com:MyDuck322/DevOops.git
ну и yes при git push надо писать полностью, ну типа ответственно чтобы видимо подходить.
